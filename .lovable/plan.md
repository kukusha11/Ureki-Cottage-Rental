

# Booking Management System with iCal Sync

## Overview

Build a simple admin page where your mother can log phone reservations, stored in a Lovable Cloud database, with an iCal feed endpoint that Booking.com can import to keep availability in sync.

## Architecture

```text
Mother enters reservation → Supabase DB → Edge Function generates iCal feed
                                                    ↓
                                          Booking.com imports iCal URL
```

## What We'll Build

### 1. Enable Lovable Cloud
Set up the database and edge functions backend.

### 2. Database Tables
- **reservations** table: guest name, phone, check-in date, check-out date, cottage number, notes, created_at
- Simple auth using a single admin password (no full user system needed — just a PIN/password gate)

### 3. Admin Page (`/admin`)
- Simple password gate (stored as environment secret) — enter once per session
- **Add Reservation** form: guest name, phone, check-in, check-out, cottage selection, optional notes
- **Reservations List**: calendar-style or table view showing all bookings, with ability to edit/delete
- Color-coded by status (upcoming, current, past)
- Mobile-friendly for easy phone use

### 4. iCal Feed Edge Function
- Edge function at `/functions/v1/ical-feed` that queries reservations and outputs a valid `.ics` calendar file
- Each reservation becomes a VEVENT with check-in/check-out dates
- Booking.com can import this URL to automatically block booked dates

### 5. How Booking.com Sync Works
After building, you'll:
1. Copy the iCal feed URL from the admin page
2. Go to Booking.com → Property → Rates & Availability → Sync Calendars
3. Paste the URL — Booking.com will poll it periodically to update availability

## Technical Details

- **Auth**: Simple shared password stored as a Supabase secret, validated client-side per session (localStorage flag). No user profiles needed.
- **Database**: Single `reservations` table with RLS allowing read for iCal feed (anon) and full CRUD for authenticated requests.
- **iCal**: Standard RFC 5545 format generated by edge function, publicly accessible URL.
- **UI**: Uses existing shadcn components (Table, Card, Form, Calendar, Dialog).
